Card 1 — Criar sua conta no GitHub

Amanhã, cada aluno ou grupo deve abrir uma conta gratuita no GitHub. O GitHub será usado para guardar o projeto e publicar o aplicativo.

1

Acesse o site

Abra o navegador e digite: github.com.

2

Clique em Sign up

Esse botão cria uma nova conta. Se você já tiver conta, clique em Sign in.

3

Digite seu e-mail

Use um e-mail que você consiga abrir na hora, porque o GitHub pode pedir confirmação.

4

Crie uma senha

Anote em local seguro. Não compartilhe sua senha.

5

Escolha nome de usuário

Use nome simples, sem acentos e sem espaços. Exemplo: grupo-neuro-uem.

6

Confirme o e-mail

Abra seu e-mail, procure a mensagem do GitHub e confirme a conta.

Atenção: sem conta no GitHub, o grupo não conseguirá publicar o PWA no GitHub Pages.

Card 2 — Criar o repositório do projeto

O repositório é a pasta online onde ficará o aplicativo.

  1. Clique no botão + no canto superior direito.
  2. Clique em New repository.
  3. No nome, digite: Aula5-PWA ou o nome do grupo.
  4. Marque Public.
  5. Clique em Create repository.

Card 3 — Criar as pastas do jogo

O projeto precisa ter esta estrutura. O GitHub diferencia letras maiúsculas e minúsculas.

Aula5-PWA/
├── index.html
├── style.css
├── app.js
├── relatorio.js
├── manifest.json
├── sw.js
└── assets/
    ├── img/
    │   ├── fundo-neuro.png
    │   ├── robo-neuro.png
    │   ├── capa-neuro.png
    │   ├── memoria-neuro.png
    │   ├── sudoku-neuro.png
    │   ├── cruzadas-neuro.png
    │   ├── quiz-neuro.png
    │   ├── cartas-neuro.png
    │   └── trofeu-neuro.png
    ├── audio/
    │   ├── clique.mp3
    │   ├── sucesso.mp3
    │   ├── erro.mp3
    │   ├── vitoria.mp3
    │   └── ambiente.mp3
    └── icons/
        ├── icon-192.png
        └── icon-512.png

Importante: o HTML não guarda a imagem dentro dele. Ele aponta para o caminho. Se a imagem não aparecer, confira nome, pasta e extensão.

Card 4 — Imagens que o aluno deve criar

Substitua as imagens-modelo mantendo exatamente estes nomes. Cards e capa: 1024x1024. Fundo: 1920x1080. Robô: PNG transparente.

Card 5 — Escolha o jogo do grupo

Cada grupo escolhe um jogo. O app gera um prompt específico para o jogo escolhido.

Card 6 — Prompt para gerar o HTML do jogo

Depois de escolher o jogo, copie o prompt e peça para a IA gerar o arquivo HTML do jogo.

Card 7 — Onde encontrar áudio para o jogo

Use sons curtos, suaves, com volume baixo. O áudio ajuda, mas não deve ser obrigatório.

Pixabay Sound Effects

Bom para cliques, acertos e sons leves.

Mixkit

Ótimo para sons de interface e vitória.

Freesound

Biblioteca grande; verifique licença.

Zapsplat

Muitos efeitos; pode exigir cadastro gratuito.

BBC Sound Effects

Bom para pesquisa sonora e ambientes.

Celular

O grupo pode gravar sons próprios.

assets/audio/clique.mp3     → botão
assets/audio/sucesso.mp3    → acerto
assets/audio/erro.mp3       → correção
assets/audio/vitoria.mp3    → finalização
assets/audio/ambiente.mp3   → música opcional

Card 8 — Transformar em PWA

PWA é um site que pode ser instalado como aplicativo. Ele precisa de manifest, service worker e ícones.

manifest.json

Nome, ícones, tela inicial e cor do app.

sw.js

Arquivo que permite cache e funcionamento offline.

icons

icon-192.png e icon-512.png.

Card 9 — Publicar no GitHub Pages

  1. Entre no repositório.
  2. Clique em Add file e depois Upload files.
  3. Envie todos os arquivos e a pasta assets.
  4. Clique em Commit changes.
  5. Vá em Settings.
  6. Vá em Pages.
  7. Em Branch, escolha main.
  8. Em Folder, escolha /root.
  9. Clique em Save e espere o link aparecer.

Card 10 — Testes e erros comuns

Imagem não aparece

Confira se o arquivo está em assets/img e se o nome está exatamente igual. Exemplo: memoria-neuro.png não é Memoria-Neuro.png.

Som não toca

Alguns navegadores só permitem som depois do primeiro clique. Teste após clicar em algum botão.

PDF não baixa

Abra no navegador, não dentro de pré-visualização. Confira a internet, pois a biblioteca html2pdf é carregada por CDN.

GitHub Pages mostra 404

Espere alguns minutos, confira se o arquivo se chama index.html e se Pages está usando main e /root.
