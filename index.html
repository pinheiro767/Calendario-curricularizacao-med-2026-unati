Prompt 1 — jogo básico

Crie a primeira versão funcional. Ainda não precisa ser perfeita.

Atue como desenvolvedor Front-End especializado em jogos educativos.

Crie um único arquivo HTML completo, com HTML, CSS e JavaScript integrados.

Sou aluno do 1º ano de Medicina e estou desenvolvendo um jogo para a Curricularização da Extensão, voltado à estimulação cognitiva da pessoa idosa.

Nome do jogo: [COLOQUE O NOME]
Tipo de jogo: [QUIZ / MEMÓRIA / SUDOKU / PALAVRAS CRUZADAS / BOLHAS]
Função cognitiva principal: [MEMÓRIA / ATENÇÃO / LINGUAGEM / RACIOCÍNIO / REFLEXO]

Requisitos da versão 1:
- tela inicial;
- botão iniciar;
- jogo funcionando;
- botão reiniciar;
- contador de pontos ou tentativas;
- mensagem final;
- design simples;
- responsivo para celular.

Não implemente ainda recursos avançados de acessibilidade.

Entregue apenas um único arquivo HTML completo e funcional.

Prompt 2 — jogo mais bonito

Melhore layout, visual e organização, mas ainda sem acessibilidade completa.

Pegue o HTML anterior e gere uma versão 2 mais bonita e organizada.

Melhore:
- layout visual;
- cores;
- espaçamento;
- botões;
- organização das telas;
- responsividade para Android, iPhone e tablet;
- animações suaves;
- aparência de aplicativo.

Mantenha o jogo funcionando.

Ainda não transforme em versão final de acessibilidade. Faça apenas uma melhoria visual e de usabilidade inicial.

Entregue apenas o HTML completo atualizado.

Prompt 3 — acessibilidade inicial

Agora começam as adaptações para pessoa idosa.

Pegue o HTML anterior e gere a versão 3 com acessibilidade inicial para pessoa idosa.

Adicione:
- fonte mínima de 24px;
- botão aumentar fonte;
- botão diminuir fonte;
- botões grandes;
- alto contraste;
- modo claro;
- modo escuro;
- instruções mais curtas;
- ícones explicativos;
- mensagens de acerto e erro mais claras;
- botão “Como jogar”.

Explique rapidamente no próprio código, por comentários, o que foi alterado.

Entregue apenas o HTML completo atualizado.

Prompt 4 — acessibilidade avançada

Adapte para baixa visão, tremor, dificuldade motora, pessoa surda e iniciantes.

Pegue o HTML anterior e gere a versão 4 com acessibilidade avançada.

Atue como especialista em:
- Gerontologia;
- Acessibilidade Digital;
- Neuropsicologia;
- UX para pessoa idosa.

Adicione:
- tutorial passo a passo;
- botão Ajuda;
- botão Voltar ao início;
- área de toque ampliada;
- prevenção de clique acidental;
- confirmação antes de reiniciar;
- feedback visual forte para acerto e erro;
- mensagens motivacionais;
- tempo maior para respostas, se houver cronômetro;
- recursos para baixa visão;
- recursos para tremor e dificuldade motora;
- acessibilidade para pessoa surda, sem depender apenas de som;
- legendas ou mensagens visuais para todos os sons;
- espaço indicado para futuro vídeo em Libras.

Entregue apenas o HTML completo atualizado.

Prompt 5 — versão final

Agora o jogo deve ficar completo para apresentar à UNATI.

Pegue o HTML anterior e gere a versão 5 final do jogo.

Atue simultaneamente como:
- desenvolvedor Front-End Sênior;
- especialista em PWA;
- especialista em acessibilidade;
- professor de Medicina;
- especialista em Gerontologia;
- participante da UNATI avaliando o jogo.

Transforme o jogo em uma versão final, profissional e acessível.

Inclua:
- layout moderno estilo aplicativo;
- visual colorido e acolhedor;
- elementos 3D leves;
- bolhas ou animações suaves no fundo;
- modo claro e escuro;
- botão aumentar/diminuir fonte;
- botão ligar/desligar som;
- feedback visual e sonoro opcional;
- instruções simples;
- tutorial;
- botão ajuda;
- botão voltar ao início;
- botões grandes e arredondados;
- alto contraste;
- acessibilidade para baixa visão;
- acessibilidade para pessoa surda;
- espaço para Libras;
- prevenção de cliques acidentais;
- estatísticas do jogador;
- melhor pontuação salva no navegador;
- tela final de parabéns;
- compatibilidade com Android, iPhone, iPad e computador.

Entregue apenas um único arquivo HTML completo, funcional e pronto para teste.

Teste final de acessibilidade

Depois da versão 5, responda os critérios de teste e anexe prints.

Atue como avaliador de acessibilidade da UNATI.

Analise meu jogo final e responda:
1. Uma pessoa idosa consegue entender o que fazer?
2. A fonte está grande?
3. O contraste está adequado?
4. Os botões são fáceis de tocar?
5. O jogo funciona sem áudio para pessoa surda?
6. Há feedback visual suficiente?
7. A navegação é simples?
8. O jogo funciona em celular?
9. O jogo evita excesso de informação?
10. Quais 5 melhorias ainda seriam necessárias?

Dê uma nota de 0 a 10 para acessibilidade e justifique.
